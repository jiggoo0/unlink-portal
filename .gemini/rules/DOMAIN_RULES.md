# 🛡️ PORTAL DOMAIN RULES
**Strict Rules for `unlink-portal` Context**

## 1. Domain Isolation (การแยกขอบเขต)
- ห้ามดึงไฟล์ อ้างอิงไฟล์ หรือรันคำสั่งที่ส่งผลกระทบต่อโดเมนอื่น (`unlink-audit`, `unlink-registry`) หรือ Root Folder นอกเหนือจาก API ที่กำหนดไว้
- การรันคำสั่งทั้งหมด (เช่น `pnpm add`, `pnpm run build`) ต้องทำภายในโฟลเดอร์ `unlink-portal/` เท่านั้น

## 2. Execution Safety (ความปลอดภัยในการทำงาน)
- ห้ามแก้ไข Business Logic หลัก หรือทำการปรับเปลี่ยนสถาปัตยกรรม (Architecture) โดยไม่ได้รับคำสั่งแบบ Directive (การระบุชัดเจน) จากผู้ใช้ (นายเอ็มซ่ามากส์)
- **ห้าม** ทำการ `git commit`, `git push`, หรือรันคำสั่ง Deploy ขึ้น Vercel โดยพลการ
- การตัดสินใจที่ไม่มั่นใจ 100% ต้องหยุดและขอคำยืนยันจากผู้ใช้ก่อนเสมอ

## 3. UI/UX Mandate (กฎการออกแบบ)
- **Institutional Authority:** โครงสร้าง UI ต้องดูเป็นทางการ สะอาดตา และน่าเชื่อถือ (Trustworthy)
- **No Dark Mode Default:** หลีกเลี่ยงการใช้ธีมมืดเป็นค่าเริ่มต้น (เว้นแต่จะมีการกำหนดไว้เฉพาะเจาะจง) ห้ามใช้ดีไซน์ที่ดูดุดันหรือ "เซฟบ็อกซ์ลึกลับ"

## 4. Validation Protocol (การตรวจสอบ)
- เมื่อมีการแก้ไขโค้ดใดๆ ต้องรันการตรวจสอบคุณภาพภายในโฟลเดอร์นี้เสมอ
- คำสั่งบังคับ: `cd unlink-portal && pnpm run aipc` (หรือ `pnpm run check`)
- งานจะสมบูรณ์ได้ก็ต่อเมื่อผ่านการตรวจสอบแบบ 0% Error เท่านั้น
